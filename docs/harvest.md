## 技術的な判断

### ADR-001: テンプレート保存戦略
- **決定**: 静的Go構造体（DBではなく）
- **理由**: MVP段階で15種固定。DB依存排除で起動高速化・テスト容易
- **評価**: 正解。テンプレートの追加変更がなかったためDBは不要だった

### ADR-002: Canvas API for Preview
- **決定**: Canvas API（SVG/DOMではなく）
- **理由**: PDF出力とのレイアウト一貫性。パフォーマンス（200ms以内）
- **評価**: 正解。リアルタイムプレビューの描画が高速で、PDF出力結果との差異が小さい

## うまくいったこと
1. **Issue → PR → Merge のフロー確立**: 全5 PRが一発でCI通過しマージ
2. **テスト先行**: 各機能にテストを同時実装し、CI上でも全パス

## 改善できること
1. **日本語フォントのPDF埋め込み**: 現在はHelveticaフォールバック。go-fpdf + TTF埋め込みが必要
2. **E2Eテスト未実装**: コンポーネント単体テストのみ。Playwright等による統合テストが望ましい
3. **PostgreSQL接続未実装**: スキーマはあるがDB接続コードなし。MVP範囲外として適切だが次フェーズで必要
4. **画像・ロゴ対応**: 店舗ロゴのアップロード機能は将来拡張
