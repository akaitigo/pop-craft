# ADR-001: テンプレートの保存戦略 — 静的Go構造体 vs DB

## ステータス
Accepted

## コンテキスト
POPテンプレート（3業態×5パターン=15種）をどこに保存するか決定する必要がある。

### 選択肢

**A: 静的Go構造体（インメモリ）**
- テンプレート定義をGoコード内にハードコード
- 起動時にメモリにロード、DB不要

**B: PostgreSQLテーブル**
- templatesテーブルに格納
- CRUD可能、動的追加可能

**C: JSONファイル**
- テンプレートをJSONで管理
- 起動時にロード

## 決定
**A: 静的Go構造体** を採用する。

## 理由
1. **MVP段階で15テンプレートは固定** — 動的追加は将来機能（スコープ外）
2. **DBへの依存を減らす** — テンプレート取得にDB接続不要、起動が高速
3. **型安全** — GoコンパイラがテンプレートデータのIの整合性を保証
4. **テスト容易** — DB fixture不要で単体テスト可能
5. **移行は容易** — 将来DB化する場合、repository interfaceを挟めば切替可能

## 結果
- テンプレートの追加・変更はコード変更+デプロイが必要
- カスタムテンプレート機能実装時にDB化を検討
- `template.All()` / `template.ByCategory()` / `template.ByID()` のインターフェースは維持
