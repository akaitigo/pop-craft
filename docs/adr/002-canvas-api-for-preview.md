# ADR-002: POP プレビュー描画に Canvas API を採用

## ステータス
Accepted

## コンテキスト
POPのリアルタイムプレビューをブラウザ上でどう描画するか。

### 選択肢

**A: Canvas API**
- 2Dコンテキストで直接描画
- ピクセル単位の制御が可能

**B: SVG**
- XML宣言的に構造を記述
- DOMとして操作可能

**C: HTML/CSS DOM**
- divとスタイルで表現
- 最もWeb的なアプローチ

## 決定
**A: Canvas API** を採用する。

## 理由
1. **PDF出力との一貫性** — Canvas描画はPDF生成のレイアウトロジックに近く、プレビューと最終出力のズレが小さい
2. **パフォーマンス** — 入力変更のたびに全体を再描画しても200ms以内で完了（DOM再計算不要）
3. **印刷レイアウト精度** — mm単位のサイズ指定がCanvas座標系と自然に対応
4. **フォントレンダリング** — Canvas.fillTextでフォント指定可能、Web Fontと組み合わせ可能
